# 1. 学 Node 要达到什么程度？

不要以「看完教程」为标准，以能独立交付一个小型业务系统为标准。

## 必须掌握（面试 + 干活都够用）

### 语言与运行时

- 异步：Promise、async/await、错误处理
- 模块化、环境变量、基础调试
- TypeScript 在 Node 里的使用（你前端已有基础，这是优势）

### Web 框架（选一个深挖）

- 推荐：NestJS（结构清晰，接近企业后端）或 Express + 自己分层（轻量，适合先理解原理）
- 路由、中间件、请求校验、统一错误处理、日志

### 数据层

- PostgreSQL 或 MySQL 二选一（会 SQL：CRUD、联表、索引基本概念）
- ORM：Prisma 或 TypeORM（会建模、迁移、事务基础）
- Redis：缓存、session、简单分布式锁概念

### 工程能力

- RESTful API 设计（资源、状态码、分页、幂等）
- JWT / Session 鉴权、RBAC 基础
- 文件上传、定时任务、消息队列（先了解 RabbitMQ/Redis 队列即可）
- 单元测试 + 接口测试（Jest + supertest）
- Docker 跑起「前端 + API + DB + Redis」
- 部署到一台云服务器或 Railway / Render 一类平台

## 不要求现阶段掌握

- 内核级性能优化、复杂微服务治理、K8s 深度、手写底层中间件

## 能力自检（达标标准）

你能独立完成并讲清楚：

- 用户注册登录（密码加密、JWT、刷新 token 可选）
- 一个核心业务模块（如订单/文章/待办）的 CRUD + 权限
- 列表分页、搜索、软删除
- 接口文档（Swagger/OpenAPI）
- 上线后可访问，有基本监控/日志

达到这个程度，简历上可以诚实写：「具备 Node 全栈独立开发与部署能力」，而不是「学过 Node」。

---

## Node 12 周学习规划（业余 10–15h/周）

### 第 1–2 周：后端基础概念（用 Node 打通）

- 目标：理解 HTTP、REST、中间件、错误处理
- 学：HTTP 方法、状态码、Header、Cookie、CORS
- 做：Express 或 Nest 最小 API（3–5 个接口）
- 产出：Postman/Apifox 集合 + README 说明接口
- 验收：能解释「一次请求从进入到响应」完整链路

### 第 3–4 周：数据库 + ORM

- 目标：会建模、会写业务 SQL
- 学：表设计、一对多/多对多、索引入门、事务
- 做：用户表 + 业务表（如订单/文章）+ 关联查询
- 产出：ER 图 + migration 文件
- 验收：能独立完成「创建订单并扣库存」这类简单事务（哪怕库存先写假逻辑）

### 第 5–6 周：鉴权与权限

- 目标：这是全栈面试高频点
- 学：bcrypt、JWT、refresh token（了解即可）、RBAC
- 做：注册/登录/登出、受保护路由、角色权限（user/admin）
- 产出：前端 React 登录页对接真实 API
- 验收：能说清「token 放哪、怎么刷新、怎么防 CSRF 基础思路」

### 第 7–8 周：工程化

- 目标：从「能跑」到「像工程师写的」
- 学：分层（controller/service/repository）、DTO 校验、统一异常、日志
- 做：Swagger 文档、Jest 测试、环境配置（dev/prod）
- 产出：代码结构清晰，有 5–10 个测试用例

### 第 9–10 周：Redis + 常见业务

- 目标：补全真实业务常见件
- 学：缓存策略、限流入门、任务队列概念
- 做：热点列表缓存、接口限流、异步任务（如发送邮件 mock）
- 产出：能解释「为什么用 Redis、缓存穿透/击穿基本概念」

### 第 11–12 周：部署 + 复盘项目

- 目标：形成简历项目
- 做：Docker Compose 一键启动；部署到云
- 写：项目文档（架构图、技术选型、难点 3 条）
- 模拟面试：项目介绍 3 分钟 + 深挖问答 10 题

### 最终交付物（简历用）

- 例如：「任务协作平台 / 轻量电商后台 / 知识库管理」
- 技术栈：React + NestJS + PostgreSQL + Redis + Docker
